## القصة — لماذا EurekAI؟

وُلِد **EurekAI** خلال [Mistral AI Worldwide Hackathon](https://luma.com/mistralhack-online) ([الموقع الرسمي](https://worldwide-hackathon.mistral.ai/)) (مارس 2026). كنت بحاجة إلى موضوع — وجاءت الفكرة من شيء عملي جدًا: فأنا أستعد بانتظام للاختبارات مع ابنتي، وفكرت أنه لا بد من إمكانية جعل ذلك أكثر مرحًا وتفاعلية بفضل الذكاء الاصطناعي.

الهدف: أخذ **أي مدخل** — صورة للدرس، نص منسوخ ولصق، تسجيل صوتي، بحث ويب — وتحويله إلى **ملخصات مراجعة، بطاقات تعليمية، اختبارات، بودكاست، نصوص ذات فراغات، رسومات توضيحية، وأكثر**. كل ذلك مدعوم بنماذج Mistral AI الفرنسية، مما يجعله حلًا مناسبًا طبيعيًا للطلاب الناطقين بالفرنسية.

تم تصميم [النموذج الأولي](https://github.com/jls42/worldwide-hackathon.mistral.ai) في 48 ساعة أثناء الهاكاثون كإثبات مفهوم حول خدمات Mistral — كان يعمل بالفعل، لكنه محدود. ومنذ ذلك الحين، أصبح EurekAI مشروعًا حقيقيًا: نصوص ذات فراغات، التنقل داخل التمارين، استخراج محتوى الويب، رقابة أبوية قابلة للتهيئة، مراجعة متعمقة للكود، وأكثر من ذلك بكثير. يتم توليد كامل الكود بواسطة الذكاء الاصطناعي — أساسًا [Claude Code](https://code.claude.com/)، مع بعض المساهمات عبر [Codex](https://openai.com/codex/) و[Gemini CLI](https://geminicli.com/).

## تعمق — الميزات

### إدخال متعدد الوسائط

يقبل EurekAI 4 أنواع من المصادر، مع رقابة بحسب الملف الشخصي (مفعلة افتراضيًا للأطفال والمراهقين) :

- **استيراد الملفات** — ملفات JPG وPNG أو PDF تتم معالجتها بواسطة `mistral-ocr-latest` (نص مطبوع، جداول، كتابة يدوية)، أو ملفات نصية (TXT، MD) تُستورد مباشرة. تستخدم عمليات الرفع المتعددة نظام **جلسات الرفع**: تقدم فردي لكل ملف، إعادة محاولة للملف الذي فشل دون إعادة إرسال البقية، وإغلاق الجلسة عند الانتهاء. يكشف OCR عن **درجة ثقة** لكل صفحة (`overall` + `perPage[]`)، وتُعرض في الواجهة على شكل شارة من المستوى `high` / `medium` / `low` (عتبات ~0.9 / ~0.7) — تنبيه دون حظر إذا كان المسح بجودة منخفضة.
- **نص حر** — اكتب أو الصق أي محتوى. يُراقَب قبل التخزين إذا كانت الرقابة مفعلة.
- **إدخال صوتي** — سجّل صوتًا في المتصفح. تتم عملية النسخ بواسطة `voxtral-mini-latest`. يتيح الإعداد `language="fr"` تحسين التعرف.
- **ويب / URL** — الصق عنوان URL واحدًا أو أكثر لاستخراج المحتوى مباشرة (Readability + Lightpanda للصفحات المعتمدة على JavaScript)، أو اكتب كلمات مفتاحية للبحث عبر الويب بواسطة Agent Mistral. الحقل الواحد يقبل الاثنين — تُفصل عناوين URL والكلمات المفتاحية تلقائيًا، وينشئ كل نتيجة مصدرًا مستقلًا.

### توليد المحتوى بالذكاء الاصطناعي

سبعة أنواع من مواد التعلم المولدة:

| المولد | النموذج | المخرجات |
|---|---|---|
| **ملخص مراجعة** | `mistral-large-latest` | عنوان، ملخص، نقاط رئيسية، مفردات، اقتباسات، طرَفة |
| **بطاقات تعليمية** | `mistral-large-latest` | بطاقات سؤال/جواب مع مراجع إلى المصادر (عدد قابل للتهيئة) |
| **اختبار اختيار من متعدد** | `mistral-large-latest` | أسئلة متعددة الخيارات، شروحات، مراجعة تكيفية (عدد قابل للتهيئة) |
| **نصوص ذات فراغات** | `mistral-large-latest` | جمل لإكمالها مع تلميحات، تحقق متسامح (Levenshtein) |
| **بودكاست** | `mistral-large-latest` + Voxtral TTS | نص بصوتين → صوت MP3 |
| **رسم توضيحي** | Agent `mistral-large-latest` | صورة تعليمية عبر الأداة `image_generation` |
| **اختبار صوتي** | `mistral-large-latest` + Voxtral TTS + STT | أسئلة TTS → إجابة STT → تحقق بالذكاء الاصطناعي |

### مدرس ذكاء اصطناعي عبر الدردشة

مدرس حواري مع وصول كامل إلى ملفات الدروس:

- يستخدم `mistral-large-latest`
- **استدعاء الأدوات**: يمكنه إنشاء ملخصات أو بطاقات تعليمية أو اختبارات أو نصوص ذات فراغات أثناء المحادثة
- سجل من 50 رسالة لكل درس
- رقابة على المحتوى إذا كانت مفعلة للملف الشخصي

### الموجّه التلقائي

يستخدم الموجّه `mistral-small-latest` لتحليل محتوى المصادر واقتراح المولدات الأكثر ملاءمة من بين 7 متاحة. تعرض الواجهة التقدم في الوقت الحقيقي: أولًا مرحلة تحليل، ثم عمليات التوليد الفردية مع إمكانية الإلغاء.

### التعلم التكيفي

- **إحصاءات الاختبارات**: تتبع المحاولات والدقة لكل سؤال
- **مراجعة الاختبار**: تنشئ 5-10 أسئلة جديدة تستهدف المفاهيم الضعيفة
- **كشف التعليمات**: يكتشف تعليمات المراجعة ("أعرف درسي إذا كنت أعرف...") ويمنحها الأولوية في المولدات النصية المتوافقة (ملخص، بطاقات تعليمية، اختبار، نصوص ذات فراغات)

### الأمان والرقابة الأبوية

- **4 فئات عمرية**: طفل (≤10 سنوات)، مراهق (11-15)، طالب (16-25)، بالغ (26+)
- **رقابة المحتوى**: `mistral-moderation-latest` مع 10 فئات متاحة، 5 محظورة افتراضيًا للأطفال/المراهقين (`sexual`، `hate_and_discrimination`، `violence_and_threats`، `selfharm`، `jailbreaking`). الفئات قابلة للتخصيص حسب الملف الشخصي في الإعدادات.
- **PIN أبوي**: تجزئة SHA-256، مطلوب للملفات الشخصية الأقل من 15 سنة. في النشر الإنتاجي، يُنصح باستخدام تجزئة بطيئة مع ملح (Argon2id، bcrypt).
- **قيود الدردشة**: دردشة الذكاء الاصطناعي معطلة افتراضيًا لمن هم دون 16 عامًا، ويمكن للأهل تفعيلها

### نظام متعدد الملفات الشخصية

- ملفات شخصية متعددة مع الاسم والعمر والصورة الرمزية وتفضيلات اللغة
- **صوت حسب الملف الشخصي** (`Profile.mistralVoices?: { host, guest }`) — يمكن لكل طفل أن يمتلك زوجًا خاصًا من أصوات البودكاست/الاختبار الصوتي
- **سمة حسب الملف الشخصي** (`Profile.theme: 'dark' | 'light'`) — تبديل تلقائي عند تغيير الملف الشخصي، محفوظ في جهة الخلفية
- ترتبط المشاريع بالملفات الشخصية عبر `profileId`
- الحذف المتسلسل: حذف ملف شخصي يحذف جميع مشاريعه

### تتبع تكاليف API

يتم قياس كل استدعاء لـ Mistral (الدردشة، OCR، STT، TTS، الرقابة، الوكلاء) لتوفير تقدير شفاف باليورو (€) للمستخدم — دون مفاجآت في الفوترة.

- **مصدر الحقيقة**: `helpers/pricing.ts` — `MODEL_PRICING` حسب بادئة النموذج (مثال: `mistral-large` → إدخال 0.5 €/مليون رمز، إخراج 1.5 €/مليون رمز)، `PRICING_SOURCES` مع عناوين URL لوثائق Mistral لإعادة الكشط الدوري
- **الوحدات المدعومة**: `tokens`، `characters` (TTS)، `pages` (OCR)، `audio-seconds` (STT) — التحويل مدفوع بواسطة `helpers/cost-calc.ts`
- **سلسلة القياس**: `helpers/tracked-client.ts` (تغليف عميل Mistral) → `helpers/usage-context.ts` (AsyncLocalStorage) → `helpers/cost-calc.ts` → `helpers/cost-persist.ts` → `helpers/cost-middleware.ts` (حقن في استجابة HTTP)
- **الواجهة**: شارة التكلفة لكل عملية توليد (`src/partials/cost-badge-gen.html`)، ولكل مصدر (`cost-badge-src.html`)، والإجمالي التراكمي في لوحة التحكم (`Project.totalCost`)
- **نقاط النهاية**: الاستجابات `/generate/*` و`/sources/*` تتضمن `costDelta: { estimatedEuros, perModel }`. تعيد `GET /projects/:pid` `totalCost` + `costLog[]` من السجل. تعرض `GET /api/config/status` `costEstimateAvailable`

### TTS متعدد المزودين وأصوات مخصصة

- **Mistral Voxtral TTS** (الافتراضي): `voxtral-mini-tts-latest`، لا حاجة إلى مفتاح إضافي
- **ElevenLabs** (بديل): `eleven_v3`، أصوات طبيعية، يتطلب `ELEVENLABS_API_KEY`
- المزود قابل للتهيئة في إعدادات التطبيق
- **أصوات مخصصة**: يمكن للأهل إنشاء أصواتهم الخاصة عبر واجهة Mistral Voices (انطلاقًا من عينة صوتية) وتعيينها لأدوار المضيف/الضيف — وعندها تُقرأ البودكاستات والاختبارات الصوتية بصوت أحد الوالدين، مما يجعل التجربة أكثر غمرًا للطفل
- دوران صوتيان قابلان للتهيئة: **المضيف** (الراوي الرئيسي) و**الضيف** (الصوت الثاني في البودكاست)
- الكتالوج الكامل لأصوات Mistral متاح في الإعدادات، مع إمكانية التصفية حسب اللغة

### التدويل

- الواجهة متاحة بـ 9 لغات: fr، en، es، pt، it، nl، de، hi، ar
- المطالبات الذكية تدعم 15 لغة (fr، en، es، de، it، pt، nl، ja، zh، ko، ar، hi، pl، ro، sv)
- اللغة قابلة للتهيئة حسب الملف الشخصي

## النشر باستخدام حاوية

يتم نشر الصورة على **GitHub Container Registry** :

```bash
# Télécharger l'image
podman pull ghcr.io/jls42/eurekai:latest

# Lancer EurekAI
mkdir -p ./data
podman run -d --name eurekai \
  -e MISTRAL_API_KEY=<your_api_key> \
  -e ELEVENLABS_API_KEY=<your_api_key> \
  -v ./data:/app/output:U \
  -p 3000:3000 \
  ghcr.io/jls42/eurekai:latest

```

> **`:U`** هو علم Podman rootless يضبط أذونات وحدة التخزين تلقائيًا.

```bash
# Build local
podman build -t eurekai -f Containerfile .

# Publier sur ghcr.io (mainteneurs)
./scripts/publish-ghcr.sh
```

## القرارات المعمارية

| القرار | المبرر |
|---|---|
| **Alpine.js بدل React/Vue** | بصمة صغيرة جدًا، وتفاعلية خفيفة مع TypeScript المترجم بواسطة Vite. مثالي لـ hackathon حيث السرعة مهمة. |
| **الاستمرارية عبر ملفات JSON** | بدون تبعيات، وبدء فوري. لا توجد قاعدة بيانات لإعدادها — نبدأ مباشرة وانطلق. |
| **Vite + Handlebars** | أفضل ما في العالمين: HMR سريع للتطوير، وpartials HTML لتنظيم الكود، وTailwind JIT. |
| **Prompts مركزية** | جميع prompts الخاصة بالذكاء الاصطناعي في `prompts.ts` — سهل التكرار والاختبار والتكييف حسب اللغة/الفئة العمرية. |
| **نظام تعدد التوليدات** | كل توليد هو كائن مستقل بمعرّف خاص به — يتيح عدة بطاقات، اختبارات، إلخ. لكل درس. |
| **Prompts مكيّفة حسب العمر** | 4 فئات عمرية بمفردات وتعقيد ونبرة مختلفة — نفس المحتوى يُدرّس بشكل مختلف حسب المتعلم. |
| **ميزات مبنية على الوكلاء** | توليد الصور والبحث على الويب يستخدمان وكلاء Mistral مؤقتين — دورة حياة نظيفة مع تنظيف تلقائي. |
| **سحب ذكي من URL** | حقل واحد يقبل عناوين URL وكلمات مفتاحية مختلطة — تُسحب عناوين URL عبر Readability (صفحات ثابتة) مع fallback إلى Lightpanda (صفحات JS/SPA)، وتؤدي الكلمات المفتاحية إلى تشغيل وكيل Mistral web_search. كل نتيجة تنشئ مصدرًا مستقلًا. |
| **TTS متعدد المزودين** | Mistral Voxtral TTS افتراضيًا (بدون مفتاح إضافي)، وElevenLabs كخيار بديل — قابل للتهيئة دون إعادة تشغيل. |
