## القصة — لماذا EurekAI؟

وُلد **EurekAI** خلال [هاكاثون Mistral AI العالمي](https://luma.com/mistralhack-online) ([الموقع الرسمي](https://worldwide-hackathon.mistral.ai/)) (مارس 2026). كنت بحاجة إلى موضوع — وجاءت الفكرة من شيء عملي جدًا: أنا أستعد بانتظام للاختبارات مع ابنتي، وفكرت أنه لا بد من إمكانية جعل ذلك أكثر متعة وتفاعلية بفضل الذكاء الاصطناعي.

الهدف: أخذ **أي مدخل** — صورة للدرس، نص منسوخ ولصقًا، تسجيل صوتي، بحث على الويب — وتحويله إلى **ملخصات للمراجعة، بطاقات تعليمية، اختبارات، بودكاست، نصوص مع فراغات، رسومات، وأكثر من ذلك**. كل ذلك مدعوم بنماذج Mistral AI الفرنسية، مما يجعله حلًا مناسبًا طبيعيًا للطلاب الناطقين بالفرنسية.

تم تصميم [النموذج الأولي](https://github.com/jls42/worldwide-hackathon.mistral.ai) في 48 ساعة خلال الهاكاثون كإثبات مفهوم حول خدمات Mistral — كان يعمل بالفعل، لكنه محدود. ومنذ ذلك الحين، أصبح EurekAI مشروعًا حقيقيًا: نصوص مع فراغات، التنقل بين التمارين، استخراج محتوى الويب، رقابة أبوية قابلة للتهيئة، مراجعة معمقة للكود، وأكثر من ذلك بكثير. يتم توليد كامل الكود بواسطة الذكاء الاصطناعي — بشكل أساسي [Claude Code](https://code.claude.com/)، مع بعض المساهمات عبر [Codex](https://openai.com/codex/) و[Gemini CLI](https://geminicli.com/).

## نظرة معمقة — الميزات

### الإدخال متعدد الوسائط

يقبل EurekAI 4 أنواع من المصادر، مع رقابة حسب الملف الشخصي (مفعلة افتراضيًا للأطفال والمراهقين) :

- **استيراد الملفات** — ملفات JPG وPNG أو PDF تُعالج بواسطة `mistral-ocr-latest` (نص مطبوع، جداول، كتابة يدوية)، أو ملفات نصية (TXT, MD) تُستورد مباشرة. تستخدم عمليات الرفع متعددة الملفات نظام **جلسات رفع**: تقدم فردي لكل ملف، وإعادة محاولة للملف الفاشل دون إعادة إرسال الملفات الأخرى، وإغلاق الجلسة عند الانتهاء. يوفّر OCR **درجة ثقة** متوسطة (`average`، مقيدة ضمن `[0,1]`، محسوبة من `averagePageConfidenceScore` التي يعيدها Mistral)، وتُعرض في الواجهة على شكل شارة tier `high` / `medium` / `low` (الحدود ~0.9 / ~0.7) — تنبيه دون حظر إذا كان المسح منخفض الجودة.
- **نص حر** — اكتب أو الصق أي محتوى. يُخضع للرقابة قبل التخزين إذا كانت الرقابة مفعلة.
- **إدخال صوتي** — سجّل صوتًا في المتصفح. يُنسخ بواسطة `voxtral-mini-latest`. يَحسّن المعامل `language="fr"` عملية التعرّف.
- **الويب / URL** — الصق عنوان URL واحدًا أو أكثر لاستخراج المحتوى مباشرة (Readability + Lightpanda للصفحات التي تعتمد على JavaScript)، أو اكتب كلمات مفتاحية للبحث على الويب عبر Agent Mistral. الحقل الواحد يقبل الاثنين — تُفصل عناوين URL والكلمات المفتاحية تلقائيًا، ويُنشئ كل نتيجة مصدرًا مستقلًا.

### توليد المحتوى بالذكاء الاصطناعي

يتم توليد سبعة أنواع من المواد التعليمية:

| المولّد | النموذج | المخرجات |
|---|---|---|
| **ملخص للمراجعة** | `mistral-large-latest` | عنوان، ملخص، نقاط رئيسية، مفردات، اقتباسات، حكاية |
| **بطاقات تعليمية** | `mistral-large-latest` | بطاقات س/ج مع إحالات إلى المصادر (عدد قابل للتهيئة) |
| **اختبار QCM** | `mistral-large-latest` | أسئلة اختيار من متعدد، شروحات، مراجعة تكيفية (عدد قابل للتهيئة) |
| **نصوص مع فراغات** | `mistral-large-latest` | جمل لإكمالها مع تلميحات، تحقق متسامح (Levenshtein) |
| **بودكاست** | `mistral-large-latest` + Voxtral TTS | نص بصوتين → صوت MP3 |
| **رسم توضيحي** | Agent `mistral-large-latest` | صورة تعليمية عبر الأداة `image_generation` |
| **اختبار صوتي** | `mistral-large-latest` + Voxtral TTS + STT | أسئلة TTS → إجابة STT → تحقق بالذكاء الاصطناعي |

### معلم ذكاء اصطناعي عبر الدردشة

معلم حواري مع وصول كامل إلى وثائق الدروس:

- يستخدم `mistral-large-latest`
- **استدعاء الأدوات**: يمكنه توليد ملخصات أو بطاقات تعليمية أو اختبارات أو نصوص مع فراغات أثناء المحادثة
- سجل 50 رسالة لكل درس
- رقابة على المحتوى إذا كانت مفعلة للملف الشخصي

### الموجّه التلقائي

يستخدم الموجّه `mistral-small-latest` لتحليل محتوى المصادر واقتراح المولدات الأكثر ملاءمة من بين 7 متاحة. تعرض الواجهة التقدم في الوقت الحقيقي: أولًا مرحلة التحليل، ثم عمليات التوليد الفردية مع إمكانية الإلغاء.

### التعلم التكيفي

- **إحصاءات الاختبارات**: تتبع المحاولات والدقة لكل سؤال
- **مراجعة الاختبار**: تُنشئ 5-10 أسئلة جديدة تستهدف المفاهيم الضعيفة
- **اكتشاف التعليمات**: يكتشف تعليمات المراجعة ("أنا أعرف درسي إذا كنت أعرف...") ويمنحها الأولوية في المولدات النصية المتوافقة (ملخص، بطاقات تعليمية، اختبار، نصوص مع فراغات)

### الأمان والرقابة الأبوية

- **4 فئات عمرية**: طفل (≤10 سنوات)، مراهق (11-15)، طالب (16-25)، بالغ (26+)
- **رقابة المحتوى**: `mistral-moderation-latest` مع 10 فئات متاحة، 5 محظورة افتراضيًا للأطفال/المراهقين (`sexual`، `hate_and_discrimination`، `violence_and_threats`، `selfharm`، `jailbreaking`). الفئات قابلة للتخصيص حسب الملف الشخصي في الإعدادات.
- **PIN أبوي**: تجزئة SHA-256، مطلوب للملفات الشخصية لمن هم دون 15 عامًا. في النشر الإنتاجي، يُنصح باستخدام تجزئة بطيئة مع ملح (Argon2id، bcrypt).
- **قيود الدردشة**: تعطيل دردشة الذكاء الاصطناعي افتراضيًا لمن هم دون 16 عامًا، ويمكن للوالدين تفعيلها

### نظام متعدد الملفات الشخصية

- ملفات شخصية متعددة مع الاسم، العمر، الصورة الرمزية، تفضيلات اللغة
- **صوت لكل ملف شخصي** (`Profile.mistralVoices?: { host, guest }`) — يمكن لكل طفل امتلاك زوج الأصوات الخاص به للبودكاست/الاختبار الصوتي
- **سمة لكل ملف شخصي** (`Profile.theme: 'dark' | 'light'`) — تبديل تلقائي عند تغيير الملف الشخصي، محفوظة في جهة الخادم
- المشاريع مرتبطة بالملفات الشخصية عبر `profileId`
- الحذف المتسلسل: حذف ملف شخصي يحذف جميع مشاريعه

### تتبع تكاليف API

كل استدعاء لـ Mistral (الدردشة، OCR، STT، TTS، الرقابة، الوكلاء) مضبوط لتقديم تقدير € **شفاف** للمستخدم — لا مفاجآت في الفوترة.

- **مصدر الحقيقة**: `helpers/pricing.ts` — `MODEL_PRICING` حسب بادئة النموذج (مثال: `mistral-large` → input 0.5 €/M tokens, output 1.5 €/M tokens)، `PRICING_SOURCES` مع روابط توثيق Mistral لإعادة الاستخراج الدوري
- **الوحدات المدعومة**: `tokens`، `characters` (TTS)، `pages` (OCR)، `audio-seconds` (STT) — التحويل بإشراف `helpers/cost-calc.ts`
- **سلسلة القياس**: `helpers/tracked-client.ts` (wrap client Mistral) → `helpers/usage-context.ts` (AsyncLocalStorage) → `helpers/cost-calc.ts` → `helpers/cost-persist.ts` → `helpers/cost-middleware.ts` (إدراج في استجابة HTTP)
- **الواجهة**: شارة التكلفة لكل عملية توليد (`src/partials/cost-badge-gen.html`)، لكل مصدر (`cost-badge-src.html`)، والإجمالي التراكمي في لوحة التحكم (`Project.totalCost`)
- **النقاط النهائية**: الاستجابات `/generate/*` و`/sources/*` تزيّن الكائن المعاد (`estimatedCost` و`usage` و`costBreakdown`) من نوع Generation / Source. يضيف `POST /generate/auto/route` حقل `costDelta: number` لتكلفة التوجيه فقط. يعيد `GET /projects/:pid` المشروع معززًا بـ `totalCost` (المجموع المحسوب من `costLog[]`) + السجل الكامل

### TTS متعدد المزودين وأصوات مخصصة

- **Mistral Voxtral TTS** (افتراضي) : `voxtral-mini-tts-latest`، لا حاجة إلى مفتاح إضافي
- **ElevenLabs** (بديل) : `eleven_v3`، أصوات طبيعية، يتطلب `ELEVENLABS_API_KEY`
- المزود قابل للتهيئة في إعدادات التطبيق
- **أصوات مخصصة**: يمكن للوالدين إنشاء أصواتهم الخاصة عبر واجهة Mistral Voices (انطلاقًا من عينة صوتية) وإسنادها إلى دوري المضيف/الضيف — عندها تُقرأ البودكاستات والاختبارات الصوتية بصوت أحد الوالدين، مما يجعل التجربة أكثر غمرًا للطفل
- دورا الصوت قابلان للتهيئة: **المضيف** (الراوي الرئيسي) و**الضيف** (الصوت الثاني في البودكاست)
- الكتالوج الكامل لأصوات Mistral متاح في الإعدادات، مع إمكانية التصفية حسب اللغة

### التدويل

- الواجهة متاحة بـ 9 لغات: fr, en, es, pt, it, nl, de, hi, ar
- تدعم تعليمات الذكاء الاصطناعي 15 لغة (fr, en, es, de, it, pt, nl, ja, zh, ko, ar, hi, pl, ro, sv)
- اللغة قابلة للتهيئة حسب الملف الشخصي

### الاختبارات، جودة الشفرة والمساهمة

```bash

```

**خطافات Git (Husky)** : `pre-commit` يشغّل `npm test`، و`pre-push` يشغّل `npm run security`. كلاهما يمنعان commit/push عند الفشل.


بدون هذه الأدوات، يفشل `npm test` في `pretest` (لأن lizard غير موجود) ويفشل `npm run security` (لأن opengrep غير موجود). عندها تمنع خطافات husky تنفيذ commit/push.

## القرارات المعمارية

| القرار | المبرر |
|---|---|
| **Alpine.js بدلًا من React/Vue** | بصمة صغيرة جدًا، وتفاعل خفيف مع TypeScript المترجم عبر Vite. مثالي لفعالية هاكاثون حيث السرعة مهمة. |
| **الاستمرار باستخدام ملفات JSON** | لا تبعيات، وبدء فوري. لا توجد قاعدة بيانات لإعدادها — نبدأ وينطلق كل شيء. |
| **Vite + Handlebars** | أفضل ما في العالمين: HMR سريع للتطوير، وpartials HTML لتنظيم الشفرة، وTailwind JIT. |
| **مطالبات مركزية** | جميع مطالبات الذكاء الاصطناعي في `prompts.ts` — سهل التكرار والاختبار والتكييف حسب اللغة/الفئة العمرية. |
| **نظام متعدد التوليدات** | كل توليد كائن مستقل بمعرّفه الخاص — يتيح عدة بطاقات مراجعة، واختبارات، إلخ. لكل درس. |
| **مطالبات مكيّفة حسب العمر** | 4 مجموعات عمرية بمفردات وتعقيد ونبرة مختلفة — المحتوى نفسه يدرّس بشكل مختلف بحسب المتعلم. |
| **ميزات مبنية على الوكلاء** | توليد الصور والبحث على الويب يستخدمان وكلاء Mistral مؤقتين — دورة حياة نظيفة مع تنظيف تلقائي. |
| **استخراج ذكي من URL** | حقل واحد يقبل عناوين URL وكلمات مفتاحية مختلطة — تُستخرج عناوين URL عبر Readability (صفحات ثابتة) مع fallback إلى Lightpanda (صفحات JS/SPA)، وتؤدي الكلمات المفتاحية إلى تشغيل وكيل web_search من Mistral. كل نتيجة تنشئ مصدرًا مستقلًا. |
| **TTS متعدد المزوّدين** | Mistral Voxtral TTS افتراضيًا (من دون مفتاح إضافي)، وElevenLabs كبديل — قابل للتهيئة دون إعادة تشغيل. |
