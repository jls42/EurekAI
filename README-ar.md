## القصة — لماذا EurekAI؟

وُلد **EurekAI** خلال [هاكاثون Mistral AI Worldwide](https://luma.com/mistralhack-online) ([الموقع الرسمي](https://worldwide-hackathon.mistral.ai/)) (مارس 2026). كنت أحتاج إلى موضوع — وجاءت الفكرة من شيء عملي جدًا: أنا أراجع الدروس بانتظام مع ابنتي، وفكرت أنه لا بد من إمكانية جعل ذلك أكثر متعة وتفاعلية بفضل الذكاء الاصطناعي.

الهدف: أخذ **أي مدخل** — صورة للدرس، نص منسوخ وملصق، تسجيل صوتي، بحث ويب — وتحويله إلى **بطاقات مراجعة، فلاش كارد، اختبارات، بودكاست، نصوص مع فراغات، رسوم توضيحية، والمزيد**. وكل ذلك مدعوم بنماذج Mistral AI الفرنسية، مما يجعله حلًا مناسبًا بشكل طبيعي للطلاب الناطقين بالفرنسية.

تم تصميم [النموذج الأولي](https://github.com/jls42/worldwide-hackathon.mistral.ai) خلال 48 ساعة أثناء الهاكاثون كإثبات مفهوم حول خدمات Mistral — وكان يعمل بالفعل، لكنه كان محدودًا. ومنذ ذلك الحين، أصبح EurekAI مشروعًا حقيقيًا: نصوص مع فراغات، التنقل داخل التمارين، استخراج محتوى الويب، ضبط رقابة أبوية قابل للتخصيص، مراجعة متعمقة للشفرة، وأكثر من ذلك بكثير. يتم توليد كامل الشفرة بواسطة الذكاء الاصطناعي — أساسًا [Claude Code](https://code.claude.com/)، مع بعض المساهمات عبر [Codex](https://openai.com/codex/) و[Gemini CLI](https://geminicli.com/).

## تعمق — الميزات

### إدخال متعدد الوسائط

يقبل EurekAI 4 أنواع من المصادر، مع رقابة حسب الملف الشخصي (مفعلة افتراضيًا للطفل والمراهق) :

- **استيراد الملفات** — ملفات JPG أو PNG أو PDF تُعالج بواسطة `mistral-ocr-latest` (نص مطبوع، جداول، كتابة يدوية)، أو ملفات نصية (TXT, MD) تُستورد مباشرة. تستخدم عمليات الرفع المتعددة نظام **جلسات رفع**: تقدم فردي لكل ملف، إعادة المحاولة للملف الفاشل دون إعادة إرسال البقية، وإغلاق الجلسة عند الانتهاء. يعرض OCR **درجة ثقة** متوسطة (`average`، مقيدة ضمن `[0,1]`، محسوبة من `averagePageConfidenceScore` التي يعيدها Mistral)، وتُعرض في الواجهة على شكل شارة tier `high` / `medium` / `low` (عتبات ~0.9 / ~0.7) — تنبيه دون حظر إذا كان المسح بجودة سيئة.
- **نص حر** — اكتب أو الصق أي محتوى. تتم مراجعته قبل التخزين إذا كانت الرقابة مفعلة.
- **إدخال صوتي** — سجّل صوتًا في المتصفح. تتم عملية النسخ بواسطة `voxtral-mini-latest`. يساهم الإعداد `language="fr"` في تحسين التعرف.
- **الويب / URL** — الصق عنوان URL واحدًا أو عدة عناوين لاستخراج المحتوى مباشرة (Readability + Lightpanda للصفحات التي تعتمد على JavaScript)، أو اكتب كلمات مفتاحية للبحث عبر Agent Mistral. الحقل الواحد يقبل النوعين — يتم فصل عناوين URL والكلمات المفتاحية تلقائيًا، وكل نتيجة تنشئ مصدرًا مستقلًا.

### توليد المحتوى بالذكاء الاصطناعي

سبعة أنواع من مواد التعلم المولدة:

| المولد | النموذج | الناتج |
|---|---|---|
| **بطاقة مراجعة** | `mistral-large-latest` | عنوان، ملخص، نقاط أساسية، مفردات، اقتباسات، طريفة |
| **فلاش كارد** | `mistral-large-latest` | بطاقات سؤال/جواب مع مراجع إلى المصادر (عدد قابل للتكوين) |
| **اختبار QCM** | `mistral-large-latest` | أسئلة متعددة الخيارات، شروحات، مراجعة تكيفية (عدد قابل للتكوين) |
| **نصوص مع فراغات** | `mistral-large-latest` | جمل لإكمالها مع تلميحات، تحقق متسامح (Levenshtein) |
| **بودكاست** | `mistral-large-latest` + Voxtral TTS | نص 2 صوت → ملف MP3 صوتي |
| **رسم توضيحي** | Agent `mistral-large-latest` | صورة تعليمية عبر الأداة `image_generation` |
| **اختبار صوتي** | `mistral-large-latest` + Voxtral TTS + STT | أسئلة TTS → إجابة STT → تحقق بالذكاء الاصطناعي |

### مدرب الذكاء الاصطناعي عبر الدردشة

مدرب حواري مع وصول كامل إلى وثائق الدروس:

- يستخدم `mistral-large-latest`
- **استدعاء الأدوات**: يمكنه توليد بطاقات مراجعة، فلاش كارد، اختبارات أو نصوص مع فراغات أثناء المحادثة
- سجل لـ 50 رسالة لكل درس
- رقابة على المحتوى إذا كانت مفعلة للملف الشخصي

### الموجّه التلقائي

يستخدم الموجّه `mistral-small-latest` لتحليل محتوى المصادر واقتراح أكثر المولدات ملاءمة من بين 7 متاحة. تعرض الواجهة التقدم في الوقت الحقيقي: أولًا مرحلة التحليل، ثم عمليات التوليد الفردية مع إمكانية الإلغاء.

### التعلم التكيفي

- **إحصاءات الاختبارات**: تتبع المحاولات والدقة حسب السؤال
- **مراجعة الاختبار**: توليد 5-10 أسئلة جديدة تستهدف المفاهيم الضعيفة
- **اكتشاف تعليمات المراجعة**: يكتشف تعليمات المراجعة ("أعرف درسي إذا كنت أعرف...") ويعطيها أولوية في المولدات النصية المتوافقة (بطاقة مراجعة، فلاش كارد، اختبار، نصوص مع فراغات)

### الأمان والتحكم الأبوي

- **4 فئات عمرية**: طفل (≤10 سنوات)، مراهق (11-15)، طالب (16-25)، بالغ (26+)
- **رقابة المحتوى**: `mistral-moderation-latest` مع 10 فئات متاحة، 5 منها محجوبة افتراضيًا للطفل/المراهق (`sexual`, `hate_and_discrimination`, `violence_and_threats`, `selfharm`, `jailbreaking`). الفئات قابلة للتخصيص حسب الملف الشخصي في الإعدادات.
- **PIN أبوي**: تجزئة SHA-256، مطلوب للملفات الشخصية الأقل من 15 عامًا. في النشر الإنتاجي، يُنصح باستخدام تجزئة بطيئة مع ملح (Argon2id, bcrypt).
- **قيود الدردشة**: الدردشة بالذكاء الاصطناعي معطلة افتراضيًا لمن هم دون 16 عامًا، ويمكن تفعيلها من قبل الوالدين

### نظام متعدد الملفات الشخصية

- ملفات شخصية متعددة مع الاسم، العمر، الصورة الرمزية، وتفضيلات اللغة
- **صوت حسب الملف الشخصي** (`Profile.mistralVoices?: { host, guest }`) — يمكن لكل طفل أن يملك زوجًا خاصًا من أصوات بودكاست/اختبار صوتي
- **ثيم حسب الملف الشخصي** (`Profile.theme: 'dark' | 'light'`) — تبديل تلقائي عند تغيير الملف الشخصي، ومحفوظ في جهة الخادم
- المشاريع مرتبطة بالملفات الشخصية عبر `profileId`
- حذف متسلسل: حذف ملف شخصي يحذف جميع مشاريعه

### تتبع تكاليف API

كل استدعاء Mistral (دردشة، OCR، STT، TTS، رقابة، وكلاء) مُجهز لتقديم تقدير € **شفاف** للمستخدم — دون أي مفاجآت في الفوترة.

- **مصدر الحقيقة**: `helpers/pricing.ts` — `MODEL_PRICING` حسب بادئة النموذج (مثلًا: `mistral-large` → input 0.5 €/M tokens, output 1.5 €/M tokens)، `PRICING_SOURCES` مع روابط وثائق Mistral لإعادة الجلب الدوري
- **الوحدات المدعومة**: `tokens`، `characters` (TTS)، `pages` (OCR)، `audio-seconds` (STT) — التحويل موجّه بواسطة `helpers/cost-calc.ts`
- **سلسلة القياس**: `helpers/tracked-client.ts` (wrap client Mistral) → `helpers/usage-context.ts` (AsyncLocalStorage) → `helpers/cost-calc.ts` → `helpers/cost-persist.ts` → `helpers/cost-middleware.ts` (إدراج في الاستجابة HTTP)
- **الواجهة**: شارة تكلفة لكل توليد (`src/partials/cost-badge-gen.html`)، لكل مصدر (`cost-badge-src.html`)، والإجمالي المتراكم في لوحة التحكم (`Project.totalCost`)
- **نقاط النهاية**: الاستجابات `/generate/*` و`/sources/*` تزيّن الكائن المُعاد (Generation / Source) بـ `estimatedCost` و`usage` و`costBreakdown`. يضيف `POST /generate/auto/route` حقل `costDelta: number` لتكلفة التوجيه فقط. يعيد `GET /projects/:pid` المشروع مُعزّزًا بـ `totalCost` (المجموع المحسوب من `costLog[]`) + السجل الكامل

### TTS متعدد المزودين وأصوات مخصصة

- **Mistral Voxtral TTS** : `voxtral-mini-tts-latest`، تحويل نص إلى كلام 100% من Mistral، دون الحاجة إلى مفتاح إضافي
- **أصوات مخصصة**: يمكن للوالدين إنشاء أصواتهم الخاصة عبر واجهة Mistral Voices API (انطلاقًا من عينة صوتية) وإسنادها إلى دوري المضيف/الضيف — عندها تُقرأ البودكاست والاختبارات الصوتية بصوت أحد الوالدين، مما يجعل التجربة أكثر غمرًا للطفل
- دورين صوتيين قابلين للتكوين: **المضيف** (الراوي الرئيسي) و**الضيف** (الصوت الثاني في البودكاست)
- كتالوج كامل لأصوات Mistral متاح في الإعدادات، مع إمكانية التصفية حسب اللغة

### التدويل

- الواجهة متاحة بـ 9 لغات: fr, en, es, pt, it, nl, de, hi, ar
- المطالبات الخاصة بالذكاء الاصطناعي تدعم 15 لغة (fr, en, es, de, it, pt, nl, ja, zh, ko, ar, hi, pl, ro, sv)
- اللغة قابلة للتكوين حسب الملف الشخصي

### الاختبارات، جودة الكود، والمساهمة


بدون هذه الأدوات، يفشل `npm test` عند `pretest` (لأن lizard غير موجود) ويفشل `npm run security` (لأن opengrep غير موجود). عندها تمنع خطافات husky عملية commit/push.

## القرارات المعمارية

| القرار | المبرر |
|---|---|
| **Alpine.js بدل React/Vue** | بصمة صغيرة جدًا، وتفاعلية خفيفة مع TypeScript مُجمّع بواسطة Vite. مثالي لـ hackathon حيث السرعة مهمة. |
| **الاستمرارية عبر ملفات JSON** | لا تبعيات، وبدء فوري. لا توجد قاعدة بيانات لإعدادها — نبدأ وينطلق كل شيء. |
| **Vite + Handlebars** | أفضل ما في العالمين: HMR سريع للتطوير، وpartials HTML لتنظيم الكود، وTailwind JIT. |
| **مطالبات مركزية** | جميع مطالبات الذكاء الاصطناعي داخل `prompts.ts` — سهل التكرار والاختبار والتكييف حسب اللغة/الفئة العمرية. |
| **نظام متعدد التوليدات** | كل توليد كائن مستقل بمعرّفه الخاص — يتيح عدة ملخصات، وبطاقات، واختبارات، إلخ. لكل درس. |
| **مطالبات مكيّفة حسب العمر** | 4 مجموعات عمرية مع مفردات وتعقيد ونبرة مختلفة — نفس المحتوى يعلّم بشكل مختلف بحسب المتعلم. |
| **وظائف مبنية على الوكلاء** | توليد الصور والبحث على الويب يستخدمان وكلاء Mistral مؤقتين — دورة حياة نظيفة مع تنظيف تلقائي. |
| **استخراج ذكي من URLs** | حقل واحد يقبل URLs وكلمات مفتاحية مختلطة — يتم استخراج URLs عبر Readability (الصفحات الثابتة) مع fallback إلى Lightpanda (صفحات JS/SPA)، بينما تؤدي الكلمات المفتاحية إلى تشغيل وكيل Mistral web_search. كل نتيجة تنشئ مصدرًا مستقلاً. |
| **TTS 100% Mistral** | ‏Mistral Voxtral TTS (لا حاجة إلى مفتاح إضافي beyond `MISTRAL_API_KEY`) — توليد كلام مدمج في سلسلة التكلفة وفي حل الصوت حسب اللغة. |
