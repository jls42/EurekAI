## 故事 — 为什么选择 EurekAI？

**EurekAI** 诞生于 [Mistral AI Worldwide Hackathon](https://luma.com/mistralhack-online)（[官方网站](https://worldwide-hackathon.mistral.ai/)）（2026 年 3 月）。我需要一个选题——灵感来自一件非常具体的事：我经常和女儿一起准备测验，于是我想到，也许可以借助 AI 让这件事变得更有趣、更互动。

目标是：将**任何输入**——课本照片、复制粘贴的文本、语音录音、网页搜索——转化为**复习卡片、闪卡、测验、播客、填空题、插图**等等。全部由 Mistral AI 的法语模型驱动，这使它天然适合讲法语的学生。

最初的 [原型](https://github.com/jls42/worldwide-hackathon.mistral.ai) 在黑客松期间用 48 小时完成，作为围绕 Mistral 服务的概念验证——当时已经可用，但功能有限。如今，EurekAI 已经成长为一个真正的项目：填空题、练习导航、网页抓取、可配置的家长监管、深入的代码审查等等。全部代码均由 AI 生成——主要是 [Claude Code](https://code.claude.com/)，另有部分通过 [Codex](https://openai.com/codex/) 和 [Gemini CLI](https://geminicli.com/) 完成。

## 深入了解 — 功能

### 多模态输入

EurekAI 接受 4 种来源类型，并根据配置文件进行审核（默认对儿童和青少年启用）：

- **文件导入** — JPG、PNG 或 PDF 文件由 `mistral-ocr-latest` 处理（印刷体、表格、手写），或直接导入文本文件（TXT、MD）。多文件上传使用**上传会话**系统：单文件进度、对失败文件重试而无需重新提交其他文件、完成后可关闭会话。OCR 提供一个平均化的**置信度分数**（`average`，限制在 `[0,1]` 内，根据 Mistral 返回的 `averagePageConfidenceScore` 计算），在 UI 中以第三方徽章 `high` / `medium` / `low` 显示（阈值约 0.9 / 0.7）——在扫描质量差时给出警告但不阻止。
- **自由文本** — 输入或粘贴任意内容。若审核已启用，则在存储前进行审核。
- **语音输入** — 在浏览器中录制音频。由 `voxtral-mini-latest` 转写。`language="fr"` 参数可优化识别。
- **Web / URL** — 粘贴一个或多个 URL 以直接抓取内容（对于 JS 页面使用 Readability + Lightpanda），或输入关键词通过 Mistral 代理进行网页搜索。单一输入字段可同时接受两者——URL 和关键词会自动分离，每个结果都会创建独立来源。

### AI 内容生成

生成七种学习材料：

| 生成器 | 模型 | 输出 |
|---|---|---|
| **复习卡片** | `mistral-large-latest` | 标题、摘要、关键点、词汇、引文、趣闻 |
| **闪卡** | `mistral-large-latest` | 带来源引用的问答卡（数量可配置） |
| **选择题测验** | `mistral-large-latest` | 多项选择题、解释、自适应复习（数量可配置） |
| **填空题** | `mistral-large-latest` | 带提示、容错校验（Levenshtein）的填空句 |
| **播客** | `mistral-large-latest` + Voxtral TTS | 双声部脚本 → MP3 音频 |
| **插图** | `mistral-large-latest` 代理 | 通过 `image_generation` 工具生成教育图片 |
| **语音测验** | `mistral-large-latest` + Voxtral TTS + STT | TTS 题目 → STT 回答 → AI 校验 |

### 通过聊天的 AI 导师

一个可对课程文档进行完整访问的对话式导师：

- 使用 `mistral-large-latest`
- **工具调用**：可在对话过程中生成复习卡片、闪卡、测验或填空题
- 每门课程保留 50 条消息历史
- 若对该配置文件启用，则对内容进行审核

### 自动路由器

路由器使用 `mistral-small-latest` 分析来源内容，并在 7 个可用生成器中推荐最相关的类型。界面实时显示进度：先是分析阶段，然后是各个独立生成阶段，并支持取消。

### 自适应学习

- **测验统计**：跟踪每道题的尝试次数和准确率
- **测验复习**：生成 5-10 道针对薄弱概念的新题
- **指令检测**：检测复习指令（“如果我知道……就说明我掌握了课程”），并在兼容的文本生成器（复习卡片、闪卡、测验、填空题）中优先处理

### 安全与家长控制

- **4 个年龄组**：儿童（≤10 岁）、青少年（11-15）、学生（16-25）、成人（26+）
- **内容审核**：`mistral-moderation-latest`，提供 10 个可用类别，儿童/青少年默认屏蔽 5 类（`sexual`、`hate_and_discrimination`、`violence_and_threats`、`selfharm`、`jailbreaking`）。可在设置中按配置文件自定义类别。
- **家长 PIN**：SHA-256 哈希，15 岁以下配置文件必需。生产部署建议使用带盐的慢哈希（Argon2id、bcrypt）。
- **聊天限制**：16 岁以下默认禁用 AI 聊天，可由家长启用

### 多配置文件系统

- 多个配置文件，包含姓名、年龄、头像、语言偏好
- **按配置文件语音**（`Profile.mistralVoices?: { host, guest }`）——每个孩子都可以拥有自己的播客/语音测验语音对
- **按配置文件主题**（`Profile.theme: 'dark' | 'light'`）——切换配置文件时自动切换，并在后端持久化
- 通过 `profileId` 将项目与配置文件关联
- 级联删除：删除配置文件会删除其所有项目

### API 成本跟踪

每次 Mistral 调用（聊天、OCR、STT、TTS、审核、代理）都会被记录，以向用户提供**透明**的欧元成本估算——账单不再有惊喜。

- **唯一真实来源**：`helpers/pricing.ts` — 按模型前缀的 `MODEL_PRICING`（例如：`mistral-large` → 输入 0.5 €/M tokens，输出 1.5 €/M tokens），以及带有 Mistral 文档 URL 的 `PRICING_SOURCES`，用于定期重新抓取
- **支持单位**：`tokens`、`characters`（TTS）、`pages`（OCR）、`audio-seconds`（STT）——由 `helpers/cost-calc.ts` 驱动转换
- **埋点链路**：`helpers/tracked-client.ts`（封装 Mistral 客户端）→ `helpers/usage-context.ts`（AsyncLocalStorage）→ `helpers/cost-calc.ts` → `helpers/cost-persist.ts` → `helpers/cost-middleware.ts`（注入到 HTTP 响应）
- **UI**：每次生成的成本徽章（`src/partials/cost-badge-gen.html`）、每个来源（`cost-badge-src.html`）、仪表盘中的累计总计（`Project.totalCost`）
- **端点**：`/generate/*` 和 `/sources/*` 的响应会用 `estimatedCost`、`usage` 和 `costBreakdown` 装饰返回对象（Generation / Source）。`POST /generate/auto/route` 会为仅路由成本添加 `costDelta: number` 字段。`GET /projects/:pid` 返回带有 `totalCost` 的项目（根据 `costLog[]` 计算出的总和）+ 完整历史记录

### 多提供商 TTS 与自定义语音

- **Mistral Voxtral TTS**：`voxtral-mini-tts-latest`，100% Mistral 语音合成，无需额外密钥
- **自定义语音**：家长可以通过 Mistral Voices API（基于音频样本）创建自己的声音，并分配给主持人/嘉宾角色——此时播客和语音测验会使用家长的声音播放，让孩子的体验更加沉浸
- 两个可配置语音角色：**主持人**（主叙述者）和**嘉宾**（播客第二个声音）
- 设置中提供完整的 Mistral 语音目录，可按语言筛选

### 国际化

- 界面支持 9 种语言：fr、en、es、pt、it、nl、de、hi、ar
- AI 提示词支持 15 种语言（fr、en、es、de、it、pt、nl、ja、zh、ko、ar、hi、pl、ro、sv）
- 可按配置文件设置语言

## 快速开始

```bash
# Cloner le dépôt
git clone https://github.com/jls42/EurekAI.git
cd EurekAI

# Installer les dépendances
npm install

# Configurer les clés API
cp .env.example .env
# Éditez .env avec vos clés :
#   MISTRAL_API_KEY=<your_api_key>           (requis)
#   SONAR_TOKEN=...                          (optionnel, CI SonarCloud uniquement)

# Lancer le développement
npm run dev
# → Backend :  http://localhost:3000 (API)
# → Frontend : http://localhost:5173 (serveur Vite avec HMR)
```

> **注**：Mistral Voxtral TTS 是唯一的 TTS 提供商——除 `MISTRAL_API_KEY` 外无需额外密钥。

### 环境变量

| 变量 | 必需 | 默认值 | 作用 |
|---|---|---|---|
| `MISTRAL_API_KEY` | ✅ | — | Mistral API 密钥（聊天、OCR、STT、Voxtral TTS、代理、审核） |
| `PORT` | 可选 | `3000` | Express 后端 HTTP 端口 |
| `NODE_ENV` | 可选 | `development` | 如果 `production` → Express 从 `dist/` 提供前端（否则为 `public/`） |
| `SONAR_TOKEN` | 可选 CI | — | 仅供 GitHub Actions SonarCloud 工作流使用 |

### 测试、代码质量与贡献

```bash
npm test                # vitest (déclenche pretest : lint:complexity + lint:ci + lint:deadcode)
npm run test:coverage   # couverture vitest
npm run lint            # ESLint + typescript-eslint + sonarjs
npm run lint:fix        # auto-fix
npm run format          # prettier
npm run security        # Opengrep (SAST local) — bloque sur finding ERROR
```

**Git Hooks（Husky）**：`pre-commit` 运行 `npm test`，`pre-push` 运行 `npm run security`。二者在失败时都会阻止提交/推送。

**所需外部工具（可选，但被 `pretest` / `npm run security` 使用）**：

```bash
# Lizard (Python) pour lint:complexity (CCN > 8 sur l'allowlist)
pipx install lizard          # ou : pipx run lizard

# Opengrep (binaire standalone ~40 Mo) pour npm run security
./scripts/install-opengrep.sh   # installe dans ~/.local/bin/
```

如果没有这些工具，`npm test` 会在 `pretest` 失败（缺少 lizard），`npm run security` 会失败（缺少 opengrep）。此时 husky hooks 会阻止提交/推送。

## 使用容器部署

镜像发布在 **GitHub Container Registry**：

```bash
# Télécharger l'image
podman pull ghcr.io/jls42/eurekai:latest

# Lancer EurekAI
mkdir -p ./data
podman run -d --name eurekai \
  -e MISTRAL_API_KEY=<your_api_key> \
  -v ./data:/app/output:U \
  -p 3000:3000 \
  ghcr.io/jls42/eurekai:latest
# → http://localhost:3000
```

> **`:U`** 是一个 Podman rootless 标志，会自动调整卷权限。

```bash
# Build local
```

## 架构决策

| 决策 | 理由 |
|---|---|
| **使用 Alpine.js 而不是 React/Vue** | 体积最小，借助由 Vite 编译的 TypeScript 实现轻量响应式。非常适合速度至上的黑客松。 |
| **使用 JSON 文件持久化** | 零依赖，启动即用。无需配置数据库——直接启动即可。 |
| **Vite + Handlebars** | 两全其美：开发时 HMR 很快，HTML partials 便于组织代码，Tailwind JIT。 |
| **集中式提示词** | 所有 AI 提示词都在 `prompts.ts` 中——便于按语言/年龄组迭代、测试和调整。 |
| **多生成系统** | 每个生成内容都是一个独立对象，拥有自己的 ID——允许每门课生成多个学习卡、测验等。 |
| **按年龄定制提示词** | 4 个年龄组，拥有不同的词汇、复杂度和语气——相同内容会根据学习者以不同方式教学。 |
| **基于代理的功能** | 图像生成和网页搜索使用临时的 Mistral Agents——生命周期干净，并自动清理。 |
| **智能 URL 抓取** | 单一输入框可混合接受 URL 和关键词——URL 通过 Readability（静态页面）抓取，并在需要时回退到 Lightpanda（JS/SPA 页面）；关键词会触发 Mistral web_search Agent。每个结果都会创建一个独立来源。 |
| **100% Mistral TTS** | Mistral Voxtral TTS（除 `MISTRAL_API_KEY` 外无需额外密钥）——语音合成集成到成本链路中，并支持按语言进行语音解析。 |
