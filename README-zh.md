## 故事——为什么选择 EurekAI？

**EurekAI** 诞生于 [Mistral AI Worldwide Hackathon](https://luma.com/mistralhack-online)（[官方网站](https://worldwide-hackathon.mistral.ai/)）（2026 年 3 月）。我需要一个主题——灵感来自一件非常具体的事：我经常和女儿一起准备测验，于是我想到，也许可以借助 AI 让这件事变得更有趣、更互动。

目标是：把**任何输入**——课本照片、复制粘贴的文本、语音录音、网页搜索——转换成**复习卡片、闪卡、测验、播客、完形填空、插图**等等。全部由 Mistral AI 的法语模型驱动，这使它天然适合讲法语的学生。

最初的 [原型](https://github.com/jls42/worldwide-hackathon.mistral.ai) 是在黑客松期间 48 小时内围绕 Mistral 服务作为概念验证而构建的——已经可用，但功能有限。从那时起，EurekAI 发展成了一个真正的项目：完形填空、练习导航、网页抓取、可配置的家长监管、深入代码审查，以及更多功能。整个代码库都由 AI 生成——主要是 [Claude Code](https://code.claude.com/)，并辅以 [Codex](https://openai.com/codex/) 和 [Gemini CLI](https://geminicli.com/) 的一些贡献。

## 深入了解——功能

### 多模态输入

EurekAI 接受 4 种来源类型，并根据档案进行审核（儿童和青少年默认启用）：

- **文件导入** — JPG、PNG 或 PDF 文件通过 `mistral-ocr-latest` 处理（印刷体文字、表格、手写），或直接导入文本文件（TXT、MD）。多文件上传使用**上传会话**系统：每个文件独立进度，失败文件重试而不重新提交其他文件，完成后关闭会话。OCR 提供一个**平均置信度分数**（`average`，限制在 `[0,1]`，由 Mistral 返回的 `averagePageConfidenceScore` 计算得出），在 UI 中以三级 `high` / `medium` / `low` 徽章显示（阈值约 0.9 / 0.7）——扫描质量差时会警告但不会阻止。
- **自由文本** — 输入或粘贴任意内容。若启用审核，会在存储前进行审核。
- **语音输入** — 在浏览器中录制音频。由 `voxtral-mini-latest` 转录。`language="fr"` 参数可优化识别。
- **网页 / URL** — 粘贴一个或多个 URL 以直接抓取内容（针对 JS 页面使用 Readability + Lightpanda），或输入关键词通过 Mistral Agent 进行网页搜索。单个输入框同时接受两者——URL 和关键词会自动分离，每个结果都会创建独立来源。

### AI 内容生成

生成七种学习材料：

| 生成器 | 模型 | 输出 |
|---|---|---|
| **复习卡** | `mistral-large-latest` | 标题、摘要、要点、词汇、引文、趣闻 |
| **闪卡** | `mistral-large-latest` | 带来源引用的问答卡（数量可配置） |
| **选择题测验** | `mistral-large-latest` | 多项选择题、解释、自适应复习（数量可配置） |
| **完形填空** | `mistral-large-latest` | 带提示的补全句子，宽容校验（Levenshtein） |
| **播客** | `mistral-large-latest` + Voxtral TTS | 双人脚本 → MP3 音频 |
| **插图** | `mistral-large-latest` Agent | 通过 `image_generation` 工具生成教育图片 |
| **语音测验** | `mistral-large-latest` + Voxtral TTS + STT | TTS 题目 → STT 回答 → AI 验证 |

### 通过聊天的 AI 导师

一个可对课程文档进行完整访问的对话式导师：

- 使用 `mistral-large-latest`
- **工具调用**：可在对话中生成复习卡、闪卡、测验或完形填空
- 每门课程保留 50 条消息历史
- 若为档案启用，则进行内容审核

### 自动路由器

路由器使用 `mistral-small-latest` 分析来源内容，并在 7 种可用生成器中推荐最相关的选项。界面实时显示进度：先是分析阶段，然后是各个生成步骤，并且可随时取消。

### 自适应学习

- **测验统计**：跟踪每道题的尝试次数和准确率
- **测验复习**：生成 5-10 道针对薄弱概念的新题目
- **指令检测**：检测复习指令（“如果我知道……我就掌握了这一课”），并在兼容的文本生成器（复习卡、闪卡、测验、完形填空）中优先处理

### 安全与家长控制

- **4 个年龄组**：儿童（≤10 岁）、青少年（11-15 岁）、学生（16-25 岁）、成人（26+）
- **内容审核**：`mistral-moderation-latest`，提供 10 个类别，其中 5 个对儿童/青少年默认阻止（`sexual`、`hate_and_discrimination`、`violence_and_threats`、`selfharm`、`jailbreaking`）。可在设置中按档案自定义类别。
- **家长 PIN**：SHA-256 哈希，15 岁以下档案必须提供。对于生产部署，建议使用带盐的慢哈希（Argon2id、bcrypt）。
- **聊天限制**：16 岁以下默认禁用 AI 聊天，可由家长启用

### 多档案系统

- 多个档案，包含姓名、年龄、头像、语言偏好
- **按档案语音**（`Profile.mistralVoices?: { host, guest }`）——每个孩子都可以拥有自己的一对播客/语音测验声音
- **按档案主题**（`Profile.theme: 'dark' | 'light'`）——切换档案时自动切换，并在后端持久化
- 通过 `profileId` 将项目与档案关联
- 级联删除：删除档案会删除其全部项目

### API 成本跟踪

每次 Mistral 调用（聊天、OCR、STT、TTS、审核、Agent）都会被记录，以向用户提供**透明**的 € 成本估算——账单不会有意外。

- **事实来源**：`helpers/pricing.ts` — 按模型前缀的 `MODEL_PRICING`（例如：`mistral-large` → 输入 0.5 €/M token，输出 1.5 €/M token），以及带有 Mistral 文档 URL 的 `PRICING_SOURCES` 用于定期重新抓取
- **支持单位**：`tokens`、`characters`（TTS）、`pages`（OCR）、`audio-seconds`（STT）——转换由 `helpers/cost-calc.ts` 驱动
- **埋点链路**：`helpers/tracked-client.ts`（封装 Mistral 客户端）→ `helpers/usage-context.ts`（AsyncLocalStorage）→ `helpers/cost-calc.ts` → `helpers/cost-persist.ts` → `helpers/cost-middleware.ts`（注入到 HTTP 响应）
- **UI**：每次生成的成本徽章（`src/partials/cost-badge-gen.html`）、每个来源（`cost-badge-src.html`）、仪表盘中的累计总计（`Project.totalCost`）
- **端点**：`/generate/*` 和 `/sources/*` 的响应会用 `estimatedCost`、`usage` 和 `costBreakdown` 装饰返回对象（Generation / Source）。`POST /generate/auto/route` 为仅路由成本添加 `costDelta: number` 字段。`GET /projects/:pid` 返回带有 `totalCost` 的项目（从 `costLog[]` 计算得出总和）+ 完整历史

### 多提供商 TTS 与自定义声音

- **Mistral Voxtral TTS**：`voxtral-mini-tts-latest`，100% Mistral 语音合成，无需额外密钥
- **自定义声音**：家长可以通过 Mistral Voices API（基于音频样本）创建自己的声音，并分配给主持人/嘉宾角色——这样播客和语音测验就会用家长的声音朗读，让孩子的体验更加沉浸
- 两个可配置的语音角色：**主持人**（主叙述者）和**嘉宾**（播客第二个声音）
- 设置中提供完整的 Mistral 声音目录，可按语言筛选

### 国际化

- 界面支持 9 种语言：fr、en、es、pt、it、nl、de、hi、ar
- AI 提示支持 15 种语言（fr、en、es、de、it、pt、nl、ja、zh、ko、ar、hi、pl、ro、sv）
- 语言可按档案配置

## 快速开始

```bash
# Cloner le dépôt
git clone https://github.com/jls42/EurekAI.git
cd EurekAI

# Installer les dépendances
npm install

# Configurer les clés API
cp .env.example .env
# Éditez .env avec vos clés :
#   MISTRAL_API_KEY=<your_api_key>           (requis)
#   SONAR_TOKEN=...                          (optionnel, CI SonarCloud uniquement)

# Lancer le développement
npm run dev
# → Backend :  http://localhost:3000 (API)
# → Frontend : http://localhost:5173 (serveur Vite avec HMR)
```

> **注意**：Mistral Voxtral TTS 是唯一的 TTS provider——除 `MISTRAL_API_KEY` 外不需要任何额外密钥。

### 环境变量

| 变量 | 必需 | 默认值 | 作用 |
|---|---|---|---|
| `MISTRAL_API_KEY` | ✅ | — | Mistral API 密钥（聊天、OCR、STT、Voxtral TTS、代理、审核） |
| `PORT` | 可选 | `3000` | Express 后端的 HTTP 端口 |
| `NODE_ENV` | 可选 | `development` | 若 `production` → Express 从 `dist/` 提供前端（否则为 `public/`） |
| `SONAR_TOKEN` | 可选 CI | — | 仅由 GitHub Actions SonarCloud 工作流使用 |

### 测试、代码质量与贡献

```bash
npm test                # vitest (déclenche pretest : lint:complexity + lint:ci + lint:deadcode)
npm run test:coverage   # couverture vitest
npm run lint            # ESLint + typescript-eslint + sonarjs
npm run lint:fix        # auto-fix
npm run format          # prettier
npm run security        # Opengrep (SAST local) — bloque sur finding ERROR
```

**Git 钩子（Husky）**：`pre-commit` 运行 `npm test`，`pre-push` 运行 `npm run security`。若失败，两者都会阻止 commit/push。

**所需外部工具（可选，但被 `pretest` / `npm run security` 使用）**：

```bash
# Lizard (Python) pour lint:complexity (CCN > 8 sur l'allowlist)
pipx install lizard          # ou : pipx run lizard

# Opengrep (binaire standalone ~40 Mo) pour npm run security
./scripts/install-opengrep.sh   # installe dans ~/.local/bin/
```

没有这些工具时，`npm test` 会因 `pretest` 失败（缺少 lizard），而 `npm run security` 会失败（缺少 opengrep）。随后 husky 钩子会阻止 commit/push。

## 架构决策

| 决策 | 理由 |
|---|---|
| **使用 Alpine.js 而不是 React/Vue** | 体积最小，配合由 Vite 编译的 TypeScript 具有轻量级响应性。非常适合速度至关重要的黑客松。 |
| **JSON 文件持久化** | 零依赖，瞬时启动。无需配置数据库——启动即用。 |
| **Vite + Handlebars** | 两全其美：开发时 HMR 快速，HTML partials 便于代码组织，Tailwind JIT。 |
| **集中式提示词** | 所有 AI 提示词都在 `prompts.ts` 中——便于按语言/年龄组迭代、测试和调整。 |
| **多生成系统** | 每个生成项都是带有独立 ID 的对象——允许每门课程生成多个复习卡片、测验等。 |
| **按年龄调整提示词** | 4 个年龄组，词汇、复杂度和语气各不相同——同一内容会根据学习者不同而以不同方式教学。 |
| **基于代理的功能** | 图像生成和网页搜索使用临时 Mistral Agents——具备自动清理的良好生命周期管理。 |
| **智能 URL 抓取** | 单一输入字段可混合接受 URL 和关键词——URL 通过 Readability（静态页面）抓取，必要时回退到 Lightpanda（JS/SPA 页面），关键词则触发 Mistral web_search Agent。每个结果都会创建一个独立来源。 |
| **100% Mistral TTS** | Mistral Voxtral TTS（除 `MISTRAL_API_KEY` 外无需额外密钥）——将语音合成集成到成本链路和按语言的语音识别中。 |
